# 功能说明

## 1. 登录与角色

系统内置三个角色：

- 学生：使用学生端预约和签到功能
- 教室管理员：管理自习室、座位、预约记录和违约记录
- 系统管理员：拥有全部权限，包括系统参数和用户角色分配

登录成功后，前端会保存 token，并根据用户权限展示对应菜单。

## 2. 学生预约

学生可以查看所有可用座位，并按以下条件筛选：

- 关键字
- 自习室
- 靠窗
- 有插座
- 安静区

预约时需要选择：

- 开始时间
- 预约小时数

系统会校验：

- 座位是否可用
- 预约时间是否和已有预约冲突
- 预约小时数是否超过系统参数

## 3. 签到

每个自习室都有一个动态编码字段，例如 `LIB101`。

学生到达教室后，在预约记录中输入动态编码进行签到。编码正确后，预约状态变为“已签到”。

## 4. 自动违约

后端每次处理 API 请求时都会检查预约状态。

如果预约开始 15 分钟后仍未签到：

- 预约状态变为“违约取消”
- 系统释放座位
- 后台生成一条违约记录

该时间可通过系统参数扩展，目前代码中使用课程要求的 15 分钟规则。

## 5. 管理端

管理端支持：

- 添加自习室
- 添加座位
- 查看座位列表
- 查看预约记录
- 系统管理员调整参数
- 系统管理员分配用户角色

不同角色看到的功能不同，体现 RBAC 权限控制。

## 6. 智能助手

智能助手默认使用本地关键词和规则匹配回答，也可配置 MiMo API 调用外部大模型增强回答能力。API 不可用时自动降级到本地规则。

支持的典型输入：

- 今天晚上还有空座吗
- 帮我找靠窗的座位
- 帮我找有插座的座位
- 帮我找靠窗并且有插座的座位
- 我今天定了哪里的座位

系统会返回符合条件的座位或当前用户今天的预约记录。

## 7. 统计

工作台展示：

- 开放自习室数量
- 可用座位数量
- 待签到预约数量
- 已签到数量
- 违约记录数量

这些数据来自后端 `/api/stats` 接口。
