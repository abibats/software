# 预约业务逻辑说明

## 座位查询 API

`GET /api/seats`

支持以下筛选参数：
- `keyword`: 按座位编号或自习室名模糊匹配
- `room_id`: 按自习室精确筛选
- `near_window`: 筛选窗边座位 (true/false)
- `has_power`: 筛选有电源座位 (true/false)
- `quiet_zone`: 筛选静音区座位 (true/false)

返回结果包含座位信息和当前可用状态。

## 创建预约 API

`POST /api/reservations`

请求参数：
- `seat_id`: 座位ID
- `start_time`: 开始时间 (ISO格式)
- `duration_hours`: 时长 (1-4小时)

业务规则：
1. 开始时间必须是未来的整点时间
2. 时长不超过系统参数 max_hours（默认4小时）
3. 同一座位同一时间段不可重复预约（冲突检测）
4. 每个学生同时只能有一个进行中的预约

## 签到 API

`POST /api/checkin`

请求参数：
- `reservation_id`: 预约ID
- `code`: 房间当日动态验证码

业务规则：
1. 验证码必须与座位所属自习室的 daily_code 一致
2. 只有状态为"已预约"的记录可以签到
3. 签到成功后状态变为"已签到"

## 自动违约处理

系统在每次 API 请求时执行 `auto_expire()` 函数：
1. 查询所有状态为"已预约"且开始时间超过 auto_cancel_minutes（默认15分钟）的预约
2. 将这些预约状态改为"已过期"
3. 为每条过期预约自动创建违约记录

## 取消预约

`PUT /api/reservations` (action: cancel)

业务规则：
1. 只有"已预约"状态的预约可以取消
2. 已签到或已过期的预约不可取消
3. 取消后座位恢复为可用状态
