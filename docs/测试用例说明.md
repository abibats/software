# 测试与 CI/CD 说明

## 测试框架

使用 Python 标准库 `unittest`，无需安装第三方依赖。

运行命令：
```bash
cd backend
python -m unittest discover -s backend -p "test_*.py" -v
```

### 冒烟测试 (smoke_test.py)

针对已部署环境的轻量级验证：
- 登录接口可用
- 当前用户信息获取
- 统计数据返回
- 座位列表可查询

## CI/CD 流水线

### GitHub Actions 配置 (.github/workflows/ci.yml)

**触发条件：** 每次 push 和 PR

**Job 1: backend-tests**
- 运行环境：Windows + Python 3.12
- 执行后端测试套件
- 确保所有测试通过

**Job 2: smoke-test**
- 运行环境：Ubuntu
- 依赖 backend-tests 通过
- 如果配置了 SMOKE_BASE_URL，对已部署实例进行冒烟测试

## 智能助手

### 本地规则匹配（无API时的降级方案）

支持的关键词匹配：
- 空座查询 → 查询当前可用座位数量
- 窗边/电源/静音座位 → 按属性筛选
- 我的预约 → 查询个人预约列表
- 签到规则 → 返回签到流程说明
- 违约规则 → 返回违约处理规则
- 取消预约 → 返回取消操作指引

### MiMo API 集成

当配置了 MiMo API Key 时：
- 发送系统提示词 + 用户问题 + 上下文数据
- 支持同时发送可用座位信息和用户预约信息
- 支持 OpenAI 和 Anthropic 兼容格式
