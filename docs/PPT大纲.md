## 第2页：项目概述

**项目简介**
- 自习座位预约系统：面向高校自习室的在线预约管理平台
- 学生可以查询、预约、签到座位；管理员可以管理自习室、座位和用户

**技术选型**
- 前端：原生 HTML + CSS + JavaScript（无框架）
- 后端：Python 标准库 HTTP Server（无第三方框架）
- 数据库：SQLite
- 部署：华为云 ECS + Nginx 反向代理

**为什么不用框架？**
- 课程要求理解底层原理
- 展示对 HTTP 协议、REST API 设计的深入理解

## 第8页：AI 辅助开发实践

**如何引导 AI 进行增量、精准开发：**
- 每次只描述一个明确的任务（如"修复签到缺少状态校验的bug"）
- 提供上下文：相关文件路径、函数名、期望行为
- 逐次迭代：先完成功能，再修复bug，最后补充测试
- Review AI 生成的代码：理解每一行，确保符合项目架构

**示例：**
1. "在 /api/reservations POST 中添加预约时间冲突检测" → 原子提交
2. "修复 auto_expire 函数读取参数错误" → 精准修复
3. "补充取消预约的测试用例" → 测试补充

## 第10页：CI/CD 流水线

**GitHub Actions 五阶段流水线：**

1. **Lint（代码检查）**— flake8 代码质量检查
2. **Build（构建验证）**— Python语法验证 + 前端文件检查 + 模块加载测试
3. **Test（自动化测试）**— 29个测试用例在 Windows 环境运行
4. **Deploy（自动部署）**— SSH部署到华为云ECS服务器
5. **Smoke Test（冒烟测试）**— 对线上服务进行健康检查

**效果：** push代码到main分支 → 自动检查 → 自动测试 → 自动部署 → 自动验证

## 第12页：系统架构

**前后端分离架构：**

```
浏览器 → Nginx(80端口) → Python HTTP Server(8000端口) → SQLite
```

**数据库设计：** 7张表
- roles（角色）、users（用户）、rooms（自习室）
- seats（座位）、reservations（预约）、violations（违规）
- parameters（系统参数）

**RBAC 权限模型：** 3级角色
- 系统管理员：全部权限
- 教室管理员：管理自习室和座位
- 学生：预约和签到
