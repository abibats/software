# 第13周开发流程与成员工作记录

项目名称：自习座位预约系统  
项目类型：Web 前后端项目  
当前阶段：第13周 Review 阶段  
小组人数：6人  
技术路线：JavaScript 前端 + Python 后端 + SQLite 数据库  

## 一、项目总体开发流程

本项目按照 Scrum 敏捷开发方式推进。小组先根据课程给出的实践项目要求进行需求分析，将“自习座位预约系统”拆分为学生端、管理端、权限控制、预约业务、智能助手、测试与 DevOps 六个方向，然后由 6 名成员分别负责不同模块。

截至第13周，本项目已经完成了主要基础功能，包括学生登录、座位查询、条件筛选、预约、签到、取消预约、管理端自习室和座位维护、RBAC 权限控制、预约记录查看、违约规则、系统参数、智能助手和项目文档。

## 六、第13周 Review 演示流程

建议现场按照以下顺序演示：

1. 使用系统管理员账号 `admin / 123456` 登录。
2. 展示工作台统计。
3. 展示管理后台，自习室和座位管理。
4. 添加一个新的自习室或座位。
5. 退出登录，使用学生账号 `student1 / 123456` 登录。
6. 展示学生账号看不到管理后台和全局违约记录。
7. 在学生预约页面筛选座位，例如选择“有插座”或“靠窗”。
8. 选择当前时间之后的整点进行预约。
9. 进入“我的预约”页面。
10. 输入教室动态编码完成签到。
11. 打开智能助手，输入“今天晚上还有空座吗”。
12. 输入“我今天预约了哪里”。
13. 再登录管理员账号，查看预约记录和统计变化。

## 七、目前存在的不足与后续计划

当前系统已经完成主要功能，但仍有可以继续完善的地方：

- 智能助手目前使用规则匹配，后续可以接入大语言模型。
- 当前没有真实邮件或微信提醒，后续可以接入邮件服务或企业微信。
- 当前使用 SQLite，后续可以升级为 MySQL。
- 当前前端使用原生 JavaScript，后续可以升级为 Vue 或 React。
- 当前部署是本地运行，后续可以在 DevCloud 中完善自动构建、自动测试和自动部署流水线。

## 八、总结

截至第13周，本项目已经完成超过一半的核心功能，并形成了较完整的 Web 前后端系统。系统覆盖了课程要求中的学生端预约、管理端 RBAC、签到与违约、智能化支持、测试和文档说明等内容，能够支持第13周 Review 的现场演示。
