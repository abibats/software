# 小组 6 人分工说明

项目名称：自习座位预约系统  
技术路线：前端 JavaScript + 后端 Python + SQLite  
端类型：Web 学生端 + Web 管理端

## 成员 A：组长 / 项目管理 / 需求分析

主要职责：

- 负责 DevCloud Scrum 项目创建和成员管理
- 负责需求拆解、用户故事编写和迭代计划
- 负责统一项目进度、Review 汇报和最终展示
- 负责协调前后端接口联调
- 负责整理项目汇报材料

交付内容：

- 用户故事列表
- 迭代计划
- 项目进度表
- Review 汇报 PPT
- 最终演示流程

## 协作方式

1. 组长在 DevCloud 创建 Scrum 项目和代码仓库。
2. 后端成员先确定接口格式和数据库字段。
3. 前端成员根据接口文档开发页面。
4. 测试成员根据用户故事编写测试用例。
5. 每个成员在独立分支开发，通过合并请求进入主分支。
6. 每次合并前至少完成一次本地运行验证。
7. 最终由组长组织演示，按“管理员添加座位 -> 学生预约 -> 学生签到 -> 查看统计和记录”的流程展示。
